## Висновки. 

### Часова складність: 
Жадібний алгоритм: Працює миттєво незалежно від суми. 
Його складність залежить лише від кількості номіналів монет O(N).
На графіку це виглядає як майже горизонтальна лінія. Оскільки кількість номіналів фіксована (6 штук), 
час виконання фактично константний ###O(1) для будь-якої суми. 

Динамічне програмування: 
Час виконання зростає лінійно разом зі збільшенням суми для видачі O(A * N), де A — сума.
На графіку видно швидкий зріст часу виконяння. Для дуже великих сум (наприклад, > 3000) цей метод буде працювати відчутно довго, 
оскільки йому потрібно розрахувати оптимальний шлях для кожної одиниці від 0 до 3000.

## Алгоритм Динамічного програмування краще підходить для нестандартних номіналів:
Як приклад [4, 3, 1] а треба видати 6, Жадібний видасть [4+1+1] тоді як ДП [3+3] що є правильним оптимумом
